Cutting it into tokens

Tokenization

May 11, 2019

Tokenizing is the process of taking a large amount of data and breaking it into small elements to be stored in variables. This data has tokens and delimiters. The delimiters separate the tokens.

Ex: Jack loves his 3 dogs
If the delimiters are whitespace then the previous example has 5 tokens:
  • Jack
  • loves
  • his
  • 3
  • dogs

Tokenization is done in two classes:String and Scanner

Scanner Class

Delimiter: In the scanner class, the delimiters are whitespace by default. To change the delimiters to something other than whitespace you can use the useDelimiters(String regex) method of Scanner class.

Tokens: The tokens are stored into variables and converted into their respective primitive data types automatically (byte, short, int, long, float, double). There are no method to convert to Char automatically.

Two types of methods to look at are the Token-based input methods and the Line-based input method

Token-based input methods:
  • The methods are: nextByte(), nextInt(), nextShort(), nextLong(), nextFloat() and nextDouble()
  • Skips any delimiters,
  • Reads token ending at delimiter,
  • Does not read delimiter after Token,
  • Value converted into byte, short, int, long, float or double for respective methods
  • Next() doesnâ€™t convert anything but returns a String.
Line based input method:
  • Method: nextLine()
  • Reads characters that start with delimiter
  • New Line character is read (not part of string returned)

Common Programming Error: Putting the line-based method after a token-based input method. Since the token-based input method does not read delimiters, it leaves the delimiter for the line-based method. Since in this case the delimiter is a whitespace, you are left with an empty string.

To fix this, put input.nextLine() prior to using the next line-based method to clear the buffer.
